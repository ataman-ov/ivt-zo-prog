# Контрольные вопросы [ОТЧЕТ]:
### 1. Способы объявления и инициализации одномерного массива
В C# существует несколько способов объявления и инициализации одномерного массива:
- Объявление и инициализация в одной строке:
```
int[] array1 = new int[] { 1, 2, 3, 4, 5 };
```
- Объявление с инициализацией без использования new:
```
int[] array2 = { 1, 2, 3, 4, 5 };
```
- Объявление массива с указанием размера и инициализация поэлементно:
```
int[] array3 = new int[5]; // Объявление массива на 5 элементов
array3[0] = 1;
array3[1] = 2;
array3[2] = 3;
array3[3] = 4;
array3[4] = 5;
```
### 2. Узнать размер (количество элементов) массива
В C# можно узнать размер массива с помощью свойства Length. Например:
```
int[] array = { 1, 2, 3, 4, 5 };
int size = array.Length; // size будет равно 5
```
### 3. Как работает оператор foreach
Оператор foreach используется для перебора элементов коллекции или массива. Он позволяет пройтись по всем элементам, не заботясь о том, как именно реализован доступ к элементам. Пример использования:
```
int[] array = { 1, 2, 3, 4, 5 };

foreach (int item in array)
{
    Console.WriteLine(item); // Выводит каждый элемент массива
}
```
В этом примере item будет принимать значение каждого элемента массива array по очереди.

### 4. Способы объявления и инициализации двумерного массива
В C# также есть несколько способов объявления и инициализации двумерного массива:
- Объявление и инициализация в одной строке:
```
int[,] array1 = new int[,] { { 1, 2 }, { 3, 4 } };
```
- Объявление с инициализацией без использования new:
```
int[,] array2 = { { 1, 2 }, { 3, 4 } };
```
- Объявление массива с указанием размеров и инициализация поэлементно:
```
int[,] array3 = new int[2, 2]; // Объявление двумерного массива 2x2
array3[0, 0] = 1;
array3[0, 1] = 2;
array3[1, 0] = 3;
array3[1, 1] = 4;
```
**Типы двумерных массивов**
В C# существуют два типа двумерных массивов:

Прямоугольные массивы (Rectangular Arrays): Это массивы, где все строки имеют одинаковое количество столбцов. Пример:
```
int[,] rectangularArray = new int[3, 4]; // 3 строки и 4 столбца
```
Непрямоугольные массивы (Jagged Arrays): Это массивы, где каждая строка может иметь разное количество столбцов. Пример:
```
int[][] jaggedArray = new int[3][];
jaggedArray[0] = new int[2]; // Первая строка имеет 2 столбца
jaggedArray[1] = new int[3]; // Вторая строка имеет 3 столбца
jaggedArray[2] = new int[1]; // Третья строка имеет 1 столбец
```
Таким образом, в C# можно легко работать с одномерными и двумерными массивами, используя различные методы их объявления и инициализации.

### 5. Синтаксис пользовательской функции. Параметры по умолчанию
В C# пользовательская функция (или метод) объявляется с указанием возвращаемого типа, имени метода и списка параметров в круглых скобках. Вот общий синтаксис:
```
returnType MethodName(parameterType1 parameterName1, parameterType2 parameterName2, ...)
{
    // Тело метода
    // Возврат значения (если returnType не void)
}
```
**Пример функции:**
```
public int Add(int a, int b)
{
    return a + b;
}
```
Параметры по умолчанию
В C# можно задавать параметры по умолчанию, что позволяет вызывать метод без указания всех аргументов. Параметры по умолчанию должны быть указаны в конце списка параметров.
```
public int Add(int a, int b = 5)
{
    return a + b;
}

// Примеры вызова
int result1 = Add(3); // Использует b = 5, результат 8
int result2 = Add(3, 10); // Использует b = 10, результат 13
```

### 6. Передача параметров в функцию. Ключевые слова params, ref и out
Передача параметров
В C# параметры могут передаваться в функции по значению (по умолчанию) или по ссылке.

- **По значению:** создается копия значения, и изменения внутри метода не влияют на оригинальное значение.
- **По ссылке:** передается ссылка на оригинальное значение, и изменения внутри метода влияют на оригинал.
**Ключевое слово params**
Ключевое слово params позволяет передавать переменное количество аргументов в метод. Эти аргументы собираются в массив.
```
public void PrintNumbers(params int[] numbers)
{
    foreach (var number in numbers)
    {
        Console.WriteLine(number);
    }
}

// Примеры вызова
PrintNumbers(1, 2, 3); // Выводит 1, 2, 3
PrintNumbers(4, 5);    // Выводит 4, 5
```
**Ключевое слово ref**
Ключевое слово ref позволяет передавать параметры по ссылке. Это означает, что изменения, внесенные в параметр внутри метода, будут отражены на оригинальном значении.
```
public void Increment(ref int number)
{
    number++;
}

// Пример вызова
int value = 5;
Increment(ref value); // value теперь равно 6
```
**Ключевое слово out**
Ключевое слово out также позволяет передавать параметры по ссылке, но требует, чтобы переменная была инициализирована внутри метода. Это полезно, когда метод должен возвращать несколько значений.
```
public void GetValues(out int a, out int b)
{
    a = 1;
    b = 2;
}

// Пример вызова
GetValues(out int x, out int y); // x = 1, y = 2
```
### 7. Что такое рекурсия? Особенности написания рекурсивных функций
**Рекурсия** — это метод, при котором функция вызывает саму себя для решения подзадачи. Рекурсивные функции обычно имеют два основных компонента:

1. **Базовый случай:** Условие, при котором функция прекращает рекурсивные вызовы. Это предотвращает бесконечную рекурсию.
2. **Рекурсивный случай:** Условие, при котором функция вызывает саму себя с измененными параметрами.
**Пример рекурсивной функции:**
```
public int Factorial(int n)
{
    if (n == 0) // Базовый случай
    {
        return 1;
    }
    else // Рекурсивный случай
    {
        return n * Factorial(n - 1);
    }
}

// Пример вызова
int result = Factorial(5); // result будет равно 120
```
**Особенности написания рекурсивных функций:**
1. **Базовый случай:** Всегда должен быть определен, чтобы избежать бесконечной рекурсии.
2. **Изменение параметров:** Каждый рекурсивный вызов должен приближать решение к базовому случаю.
3. **Потенциальные проблемы с производительностью:** Рекурсия может привести к переполнению стека, если глубина рекурсии слишком велика. В таких случаях может быть лучше использовать итеративные подходы.
